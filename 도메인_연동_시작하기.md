## 📋 지금 필요한 정보 3가지

### 1️⃣ Cloudflare Pages 프로젝트 정보

```

2. "Workers & Pages" 클릭
3. 프로젝트 목록에서 찾기 (gumi 또는 coop 관련)
4. 프로젝트 클릭하면 상단에 "프로젝트명.pages.dev" URL 표시됨

📝 현재 Pages URL: ___________________.pages.dev
```

### 2️⃣ 연동할 도메인 이름

```

예시:
- gumidlc.co.kr
- gumi-coop.kr
- 구미디지털협동조합.kr

📝 도메인 이름: ___________________
```

## 🎯 다음 단계 (정보 확인 후)

위 3가지 정보를 확인하면 다음 단계로 진행합니다:

### Step 1: Cloudflare에 도메인 추가
- Cloudflare Pages 프로젝트에 Custom Domain 추가
- DNS 레코드 정보 확인

### Step 2: DNS 설정 변경
- 도메인 등록 업체에서 CNAME 레코드 추가
- Cloudflare Pages URL을 타겟으로 설정

### Step 3: 전파 대기 및 확인
- DNS 전파 대기 (5분~48시간)
- HTTPS 자동 활성화 확인

### Step 4: D1 마이그레이션 실행
- 데이터베이스 테이블 생성
- 샘플 데이터 확인

### Step 5: 전체 기능 테스트
- 모든 페이지 정상 작동 확인
- 양식 제출 테스트
- 관리자 페이지 테스트

## 🔍 Cloudflare Pages URL 찾는 방법

### 방법 1: Cloudflare 대시보드

1. https://dash.cloudflare.com 접속
2. 왼쪽 메뉴에서 "Workers & Pages" 클릭
3. Pages 탭에서 프로젝트 찾기
4. 프로젝트 이름 확인 (예: gumi-digital-coop-website)
5. URL 형식: `프로젝트이름.pages.dev`

### 방법 2: GitHub Actions 확인

1. GitHub 저장소 접속: https://github.com/seojeongju/gumi-digital-coop-website
2. "Actions" 탭 클릭
3. 최근 배포 로그에서 Pages URL 확인

### 방법 3: Cloudflare 이메일 확인

프로젝트 생성 시 Cloudflare에서 보낸 이메일에 URL이 포함되어 있습니다.

## ⚡ 빠른 시작 팁

### 이미 Cloudflare Pages URL을 알고 계신다면:

```bash
# 예시: gumi-digital-coop.pages.dev

→ 도메인 이름만 알려주세요!
→ 바로 Cloudflare에 추가하는 방법 안내해드립니다!
```

### DNS 설정이 처음이라면:

걱정하지 마세요! 단계별로 자세히 안내해드립니다.
보통 5-10분이면 설정할 수 있습니다.

### 도메인이 아직 없다면:

1. 먼저 도메인 구매 (가비아, 후이즈 등에서 구매 가능)
2. 구매 후 바로 연동 가능
3. .kr 도메인 추천 (한국 협동조합에 적합)
