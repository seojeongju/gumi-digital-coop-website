도메인 연동을 시작하려면 아래 3가지 정보가 필요합니다:


1️⃣  Cloudflare Pages URL 확인
───────────────────────────────────────────────────────────────────
❓ 현재 배포된 Cloudflare Pages URL이 무엇인가요?

확인 방법:
1. https://dash.cloudflare.com 접속
2. 왼쪽 메뉴 "Workers & Pages" 클릭
3. Pages 탭에서 프로젝트 찾기 (gumi 또는 coop 관련)
4. 프로젝트 클릭 → 상단에 URL 표시됨

예시: gumi-digital-coop.pages.dev

📝 현재 Pages URL: ___________________________.pages.dev


2️⃣  연동할 도메인 이름
───────────────────────────────────────────────────────────────────
❓ 연동하려는 도메인 이름이 무엇인가요?

예시:
- gumidlc.co.kr
- gumi-coop.kr
- 구미디지털협동조합.kr

📝 도메인 이름: ___________________________


3️⃣  도메인 등록 업체 및 DNS 권한
───────────────────────────────────────────────────────────────────
❓ 도메인이 어디에 등록되어 있나요?

주요 업체:
☐ 가비아 (www.gabia.com)

☐ Cloudflare

☐ 아니오, 확인이 필요합니다

📝 등록 업체: ___________________________

📋 정보 확인 후 진행할 단계
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

위 3가지 정보를 확인하면 아래 순서로 진행합니다:

Step 1: Cloudflare에 도메인 추가 (5분)
        → Custom domains 메뉴에서 도메인 추가
        → DNS 레코드 정보 확인

Step 2: DNS 설정 변경 (10분)
        → 도메인 등록 업체에서 CNAME 레코드 추가
        → Cloudflare Pages URL을 타겟으로 설정

Step 3: DNS 전파 대기 (5분~48시간)
        → 보통 30분~2시간 소요
        → dnschecker.org에서 전파 상태 확인

Step 4: HTTPS 자동 활성화 확인 (자동)
        → Cloudflare가 자동으로 SSL 인증서 발급
        → 녹색 자물쇠 아이콘 확인

Step 5: D1 마이그레이션 실행 (15분)
        → Cloudflare D1 Console에서 5개 SQL 실행
        → 데이터베이스 테이블 생성

Step 6: 전체 기능 테스트 (20분)
        → 모든 페이지 로드 확인
        → 양식 제출 테스트
        → 관리자 페이지 테스트

총 소요 시간: 약 1시간 (DNS 전파 시간 제외)
