# ⚠️ 후이즈 DNS 설정 - 올바른 방법

**문제**: 현재 "사용할 네임서버 직접 입력" 방식을 선택하셨습니다.  
**해결**: "후이즈 기본 네임서버" 방식으로 변경하고 DNS 레코드를 관리해야 합니다.

---

## 🔄 올바른 설정 방법

### Step 1: 네임서버 설정 변경

현재 스크린샷에서 보이는 화면에서:

```
1. "후이즈 기본 네임서버" 라디오 버튼 선택 ◉
   (현재 "사용할 네임서버 직접 입력"이 선택되어 있음)

2. 화면 아래 "네임서버 추가하기" 버튼 클릭하지 않음

3. 설정 저장
```

### Step 2: DNS 관리 메뉴로 이동

네임서버를 "후이즈 기본 네임서버"로 변경한 후:

```
1. 도메인 관리 페이지로 돌아가기

2. gdamic.kr 도메인 찾기

3. "DNS 관리" 또는 "DNS 설정" 버튼 클릭
   (네임서버 설정이 아닌, DNS 레코드 관리)
```

---

## 📝 DNS 레코드 추가 방법

DNS 관리 페이지에서:

### 방법 1: DNS 레코드 관리 화면

```
┌─────────────────────────────────────────────────────────┐
│ DNS 레코드 관리                                           │
├─────────────────────────────────────────────────────────┤
│                                                           │
│ [ + 레코드 추가 ] 또는 [ + 새 레코드 ]                    │
│                                                           │
│ ┌─────────────────────────────────────────────────────┐ │
│ │ 레코드 타입: CNAME ▼                                 │ │
│ │ 호스트명: @                                          │ │
│ │ 값/Target: gumi-digital-coop-website.pages.dev      │ │
│ │ TTL: 3600                                            │ │
│ │ [ 저장 ]                                             │ │
│ └─────────────────────────────────────────────────────┘ │
│                                                           │
│ ┌─────────────────────────────────────────────────────┐ │
│ │ 레코드 타입: CNAME ▼                                 │ │
│ │ 호스트명: www                                        │ │
│ │ 값/Target: gumi-digital-coop-website.pages.dev      │ │
│ │ TTL: 3600                                            │ │
│ │ [ 저장 ]                                             │ │
│ └─────────────────────────────────────────────────────┘ │
│                                                           │
└─────────────────────────────────────────────────────────┘
```

## 🎯 정확한 단계별 가이드

### 1️⃣ 현재 화면에서 할 일

```
현재 보이는 "네임서버 변경 도메인" 화면에서:

❌ "사용할 네임서버 직접 입력" 선택 해제
✅ "후이즈 기본 네임서버" 라디오 버튼 클릭
✅ 페이지 하단 "저장" 또는 "변경" 버튼 클릭
```

### 2️⃣ 도메인 리스트로 돌아가기

```
네임서버 변경 완료 후:

→ "MY 서비스" 또는 "도메인 관리" 메뉴
→ "도메인 리스트"
→ gdamic.kr 찾기
```

### 3️⃣ DNS 관리 메뉴 찾기

gdamic.kr 도메인 옆에 버튼들이 있을 것입니다:

```
가능한 버튼 이름들:
- "DNS 관리"
- "DNS 설정"
- "레코드 관리"
- "호스트 설정"
- "관리" (클릭 후 하위 메뉴에서 DNS 관리 선택)
```

### 4️⃣ CNAME 레코드 추가

DNS 관리 페이지에서:

```
[ + 레코드 추가 ] 또는 [ + 새 레코드 ] 버튼 클릭

첫 번째 레코드:
┌──────────────────────────────────────┐
│ 레코드 타입: CNAME                    │
│ 호스트/이름: @                        │
│ 값/대상: gumi-digital-coop-website.pages.dev │
│ TTL: 3600                             │
│ [ 저장 ]                              │
└──────────────────────────────────────┘

두 번째 레코드:
┌──────────────────────────────────────┐
│ 레코드 타입: CNAME                    │
│ 호스트/이름: www                      │
│ 값/대상: gumi-digital-coop-website.pages.dev │
│ TTL: 3600                             │
│ [ 저장 ]                              │
└──────────────────────────────────────┘
```

## 💡 왜 네임서버 설정이 아닌가요?

### 잘못된 방법 (현재 선택하신 것):
```
"사용할 네임서버 직접 입력"
→ Cloudflare의 네임서버를 입력
→ 전체 DNS를 Cloudflare로 이관
→ 복잡하고 불필요한 방법
```

### 올바른 방법 (권장):
```
"후이즈 기본 네임서버" + DNS 레코드 관리
→ 후이즈 네임서버 유지
→ CNAME 레코드만 추가
→ 간단하고 빠른 방법
```

## ⚠️ 주의사항

### 루트 도메인 CNAME 문제

만약 DNS 관리 페이지에서 다음 오류가 발생하면:

```
❌ "루트 도메인(@)에는 CNAME을 설정할 수 없습니다"
```

**해결책:**

#### 방법 1: ANAME 레코드 사용
```
레코드 타입 선택 시:
- CNAME 대신
- ANAME 또는 ALIAS 선택
- 호스트: @
- 값: gumi-digital-coop-website.pages.dev
```

#### 방법 2: URL 포워딩
```
1. www만 CNAME 설정:
   타입: CNAME
   호스트: www
   값: gumi-digital-coop-website.pages.dev

2. 루트 도메인은 URL 포워딩:
   → 도메인 관리에서 "URL 포워딩" 또는 "도메인 포워딩" 메뉴
   → gdamic.kr → www.gdamic.kr로 리다이렉트 설정
```
