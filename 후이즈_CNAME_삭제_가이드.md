# 🎯 후이즈 CNAME 레코드 삭제 가이드

## 📍 목표
gdamic.kr (루트 도메인) CNAME 레코드만 삭제하고, www.gdamic.kr CNAME은 유지

---

## 🚀 단계별 실행

### STEP 1: DNS 호스트 관리 접속

**현재 위치**: 네임서버 고급설정 화면

**실행:**
```
[DNS 호스트 설정] 버튼 클릭
```

---

### STEP 2: CNAME 레코드 목록 확인

**예상 화면:**
```
┌──────────────────────────────────────────────────────┐
│ CNAME 레코드 신청정보                                 │
├──────────────────────────────────────────────────────┤
│ 선택 │ 호스트명          │ CNAME 레코드              │
├──────────────────────────────────────────────────────┤
│ ☐   │ www.gdamic.kr    │ gumi-digital-coop-...    │ ✅ 유지
│ ☐   │ gdamic.kr        │ gumi-digital-coop-...    │ ❌ 삭제
└──────────────────────────────────────────────────────┘
```

**확인사항:**
- 두 개의 CNAME 레코드가 표시되어야 함
- 호스트명이 다른지 확인 (www vs 루트)

---

### STEP 3: gdamic.kr (루트) 레코드 선택

**방법 A: 체크박스 선택**
```
gdamic.kr 행의 체크박스를 클릭
```

**방법 B: 변경/해지 라디오 버튼**
```
gdamic.kr 행에서 "해지" 라디오 버튼 선택
```

**⚠️ 주의:**
- **www.gdamic.kr는 절대 선택하지 마세요!**
- **gdamic.kr (또는 @, 또는 빈 칸)만 선택**

---

### STEP 4: 삭제 실행

**화면에 따라 다를 수 있음:**

**옵션 A: 신청 화면**
```
1. gdamic.kr 행 선택
2. "해지" 또는 "삭제" 옵션 선택
3. 하단 "신청한 추가" 버튼 클릭
```

**옵션 B: 직접 삭제**
```
1. gdamic.kr 행 선택
2. "삭제" 버튼 클릭
3. 확인 팝업에서 "확인" 클릭
```

---

### STEP 5: 저장 확인

**확인 메시지:**
```
"CNAME 레코드가 삭제되었습니다"
또는
"변경사항이 저장되었습니다"
```

**최종 확인:**
- DNS 호스트 관리 화면에서 CNAME 레코드 목록 다시 확인
- **www.gdamic.kr만 남아있어야 함**
- **gdamic.kr는 목록에서 사라져야 함**

## ⚠️ 문제 해결

### 문제 1: "삭제할 수 없습니다" 에러
**해결:**
- 레코드 상태 확인 (활성/비활성)
- 5분 후 다시 시도
- 고객센터 문의: 1544-2424

### 문제 2: www도 같이 삭제됨
**해결:**
- 즉시 www.gdamic.kr CNAME 레코드 다시 추가
- 값: `gumi-digital-coop-website.pages.dev`
- TTL: 3600

### 문제 3: 삭제 버튼이 안 보임
**해결:**
- 화면 스크롤 (위/아래)
- 체크박스 선택 후 하단 버튼 확인
- "변경" 탭 또는 "해지" 탭 확인

## 💡 왜 삭제해야 하나요?

### 이유:
- CNAME 레코드와 URL 포워딩은 동시 사용 불가
- DNS 표준 제약사항
- 루트 도메인은 URL 포워딩으로 www로 리다이렉트 예정

### 최종 구성:
```
www.gdamic.kr → Cloudflare Pages (CNAME)
gdamic.kr → www.gdamic.kr (URL 포워딩)
```
